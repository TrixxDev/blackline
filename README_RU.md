# BLACKLINE — стартовый пакет для Codex

Этот каталог содержит инструкции и проектную документацию для разработки соревновательного сетевого FPS с собственным движком.

## Что уже зафиксировано

- рабочее название: **BLACKLINE**;
- язык: **C++23**;
- сборка: **CMake + vcpkg manifest**;
- окно и ввод: **SDL3**;
- графика: собственный **RHI** с бэкендами Direct3D 11, Direct3D 12 и Vulkan;
- порядок реализации: **D3D11 → D3D12 → Vulkan**;
- RHI проектируется по явной модели D3D12/Vulkan, а не по ограничениям D3D11;
- шейдеры: единый HLSL-исходник, отдельная компиляция под DXBC, DXIL и SPIR-V;
- первый целевой результат: серый сетевой FPS-прототип с отдельным авторитетным сервером;
- модели и контент сначала временные, затем заменяются собственными ассетами через Blender/glTF pipeline.

## Канонический репозиторий

BLACKLINE разрабатывается только в этом репозитории:

```text
https://github.com/TrixxDev/blackline.git
```

Codex не должен создавать отдельный репозиторий, менять `origin`, работать в случайной пустой папке или переносить реализацию в другой проект.

## Как использовать

1. Клонируй канонический репозиторий:

```powershell
git clone https://github.com/TrixxDev/blackline.git
cd blackline
```

2. Скопируй в корень клона всё содержимое этого пакета, включая скрытые папки `.codex` и `.agents`.
3. Проверь remote:

```powershell
git remote get-url origin
```

Ожидаемый результат:

```text
https://github.com/TrixxDev/blackline.git
```

4. Открой именно этот каталог `blackline` в Codex.
5. Первым сообщением отправь содержимое `prompts/00_BOOTSTRAP.md` либо просто напиши:

```text
Read AGENTS.md, PROJECT_CONTEXT.md, PLANS.md and docs/ROADMAP.md. Then execute prompts/00_BOOTSTRAP.md exactly. Do not skip validation.
```

6. После завершения bootstrap используй `prompts/01_PHASE_0_FOUNDATION.md`.

## Важный принцип

Не проси Codex «сделать всю игру». Одна задача должна завершаться проверяемым артефактом: сборка, тест, запускаемое окно, корректный кадр, сетевой пакет или игровой сценарий.

Codex должен самостоятельно читать документацию, планировать изменения, запускать проверки и оставлять репозиторий в рабочем состоянии. Каждая фаза заканчивается только после выполнения критериев готовности.

## Git workflow

- Основная удалённая ссылка: `origin = https://github.com/TrixxDev/blackline.git`.
- Codex может создавать локальные ветки и коммиты только по явному заданию владельца.
- Codex не должен выполнять `git push`, force-push, менять remote или удалять ветки без явного разрешения.
- Перед каждой большой задачей Codex проверяет `git status --short --branch` и `git remote -v`.
- Незакоммиченные пользовательские изменения нельзя удалять, перезаписывать или прятать без согласования.
